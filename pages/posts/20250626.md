### 顶栏自定义
尝试自定义顶栏，但是文档中没有找到对应内容。折腾大半天后翻到了[issue#550](https://github.com/YunYouJun/valaxy/pull/550)，先前顶栏是和下图红框部分一起由valaxy.config.ts中的pages属性控制的，但几周前作者启用了nav属性替代pages来描述顶部的导航信息，文档没有一起更新。
![](https://cdn.jsdelivr.net/gh/Eofs791/catpic@main/posts/20250626/link.png)
需要自定义顶栏，在valaxy.config.ts中如下配置即可：  
```ts
nav: [  
      {  
        text: '归档',  
        link: '/archives/',  
        icon: 'i-ri-archive-line',  
      },  
      {  
        text: '分类',  
        link: '/categories/',  
        icon: 'i-ri-folder-2-line',  
      },  
      {  
        text: '标签',  
        link: '/tags/',  
        icon: 'i-ri-price-tag-3-line',  
      },  
      {  
        text: '网页收藏',  
        link: '/links/',  
        icon: 'i-ri-links-fill',  
      }  
    ],
```
## 2025-06-20
整理了一天以前在各平台发表的文章和图片。实现了GitHub Pages托管。（[GitHub Pages](https://valaxy.site/guide/deploy#github-pages)）
### 文章整理和编辑
选择使用obsidian进行文章管理和编辑，直接将pages或posts文件夹当作一个obsidian vault导入即可。需要注意的是，obsidian本地插入图片的语法与md标准语法不同，最好不要直接拖拽插入，建议手动输入。  
当然也可以直接用vscode或其他软件进行编辑。  
markdown语法可以参照这个网站：[Markdown教程](https://markdown.com.cn/)
#### 文件命名规则
博客文章链接一般为`https://博客域名/posts/xxxxxxx`，`xxxxxxx`这部分的内容就由文件名称决定。  
我一开始选择从0000开始计数，[后](#2025-06-26)改成文章创建时间，如这篇日志是在25年6月26日开始写的，我就将其命名为`20250626.md`。
### 图片整理
反复进行了三次的图片整理。   
开始使用的本地储存图片：
```
.
├── posts
│   ├── post1.md
│   ├── post2.md
│   └── img
│       ├── post1
│       │   ├── 1.jpg
│       │   └── 2.png
│       └── post2

```
但发现此方案非常占用空间，最终还是决定采用用[PicGo](https://github.com/Molunerfinn/PicGo)上传GitHub个人图床的形式。  
#### PicGo+GitHub储存图片
- 新建GitHub仓库作为图床，在PicGo的[releases](https://github.com/Molunerfinn/PicGo/releases)中下载最新版本。
- 在PicGo-图床设置-GitHub中选择相册进行配置
	- token获取：GitHub点击自己头像- `Settings`-`Developer Settings`-`Personal access tokens`-`Tokens (classic)`-`Generate new token`-`Generate new token (classic)`
	- 自定义域名可以填`https://cdn.jsdelivr.net/gh/Github用户名/图床仓库名@分支名`
### GitHub Pages托管
- 在GitHub创建以`GitHub用户名.github.io`为仓库名的仓库
- 在仓库`Settings`-`Actions`-`General`-`Workflow permissions`中勾选`Read and write permissions`
- 将整个博客项目（即包含`site.config.ts`以及`valaxy.config.ts`的那个目录）上传至仓库
- 博客项目已经自带了workflow相关配置，等待GitHub Pages部署完毕，在仓库`Settings`-`Pages`中选择gh-pages分支即可

### 基础的主题更改
执行`pnpm dev`在本地启用预览后，在页面底部点击vue图标打开devtool
![](https://cdn.jsdelivr.net/gh/Eofs791/catpic@main/posts/20250626/vuedevtools.png)

点击devtool顶部中间的按钮，选中想要更改的组件，再点击右上角按钮，打开vue源文件查看代码。
![](https://cdn.jsdelivr.net/gh/Eofs791/catpic@main/posts/20250626/vuedevtools1.png)

想要修改样式的话，在博客项目下的`components`文件夹里创建同名vue文件，输入更改后的代码，即可覆盖原样式。  

如我想删除主页的SiteLink，就在`components`文件夹下创建`YunPrologueSquare.vue`，并删除了原文件的SiteLink部分。
```vue
<div
  class="mt-4 flex-center w-72 md:w-150 m-auto gap-2"
  flex="~ wrap"
>
  <YunSiteLinkItem
    :page="{
      name: '博客文章',
      icon: 'i-ri-article-line',
      url: '/posts/',
    }"
  />
  <slot />
  <YunSiteLinkItem
    v-for="item, i in themeConfig.pages"
    :key="i" :page="item"
  />
</div>
```

需要注意的是，有些模块导入语句的文件路径需要更改。  
如`YunPrologueSquare.vue`的开头，原文件是这样的：
```ts
import { ref } from 'vue'
import { useThemeConfig } from '../../composables'
```
我在`components`文件夹创建同名vue文件覆盖原样式时，需要改成：
```ts
import { ref } from 'vue'
import { useThemeConfig } from 'valaxy-theme-yun/composables/config.ts'
```

### 部分页面组件删除
如图所示，每篇post底部都有文章前后跳转组件，yun主题在默认情况下，个人、网站简介及分类等页面也都会显示。
![](https://cdn.jsdelivr.net/gh/Eofs791/catpic@main/posts/20250626/QQ20250627-134553.png)
删除方法文档没有写，但在部分index.md的头部属性里已经包含了。  
如删除文章前后跳转组件，需要在md文件头部添加：
```md
nav: false
```
同理，删除目录组件（右侧边栏还在）：
```md
toc: false
```
删除右侧边栏：
```md
aside: false
```
删除评论组件（如果存在）：
```md
comment: false
```

### 萌萌计数器（快速使用版）
[Moe-Counter项目](https://github.com/journey-ad/Moe-Counter)   
直接进入[demo网站](https://count.getloli.com/)，下拉找到Tool部分，设置完参数点击Generate，即可生成自己的萌萌计数器链接，该链接每被刷新一次就会记一次数。将计数器链接插入页脚即可，[插入方式](https://valaxy.site/guide/custom/components#%E8%87%AA%E5%AE%9A%E4%B9%89%E8%A6%86%E7%9B%96%E4%B8%BB%E9%A2%98%E7%BB%84%E4%BB%B6)。  

计数器也可以自己部署，不过我没有研究过，需要另外搜索教程。
### 友链
打开`博客项目-pages-links-index.md`，修改头部属性`links: /links.json`  
在`博客项目-public`内创建`links.json`文件，然后按照以下格式填写友链即可：
```json
[
{
	"url": "博客URL",  
    "avatar": "头像URL",  
    "name": "博客名称",  
    "blog": "博客名称",  
    "desc": "博客描述",  
    "color": "按钮颜色"
},
{
}
]
```
