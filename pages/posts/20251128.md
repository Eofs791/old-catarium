## epel
安装NapCat时出现报错
```bash
[2025-11-27 11:43:24]: 安装epel (sudo dnf install -y epel-release)失败

# 安装epel后重新运行安装脚本，依旧报错
[2025-11-27 11:50:36]: 安装依赖 (sudo dnf install --allowerasing -y zip unzip jq curl screen procps-ng cpio nss mesa-libgbm atk at-spi2-atk gtk3 alsa-lib pango cairo libdrm libXcursor libXrandr libXdamage libXcomposite libXfixes libXrender libXi libXtst libXScrnSaver cups-libs libxkbcommon libX11-xcb mesa-dri-drivers mesa-libEGL mesa-libGL xcb-util xcb-util-image xcb-util-wm xcb-util-keysyms xcb-util-renderutil fontconfig dejavu-sans-fonts xorg-x11-server-Xvfb)失败
```

解决方案：

```bash
# 下载ALinux4兼容的epel包
sudo dnf install -y https://mirrors.aliyun.com/epel/epel-release-latest-9.noarch.rpm

# 创建新epel.repo文件指向国内镜像
sudo tee /etc/yum.repos.d/epel.repo > /dev/null << 'EOF'
[epel]
name=Extra Packages for Enterprise Linux 9 - $basearch
baseurl=https://mirrors.aliyun.com/epel/9/Everything/$basearch/
enabled=1
gpgcheck=0
EOF

# 更新缓存
sudo dnf clean all
sudo dnf makecache

# 安装依赖
sudo dnf install --allowerasing -y zip unzip jq curl screen procps-ng cpio nss mesa-libgbm atk at-spi2-atk gtk3 alsa-lib pango cairo libdrm libXcursor libXrandr libXdamage libXcomposite libXfixes libXrender libXi libXtst libXScrnSaver cups-libs libxkbcommon libX11-xcb mesa-dri-drivers mesa-libEGL mesa-libGL xcb-util xcb-util-image xcb-util-wm xcb-util-keysyms xcb-util-renderutil fontconfig dejavu-sans-fonts xorg-x11-server-Xvfb
```

## dialog
直接下载源码至服务器，然后解压安装

地址：[fossies](https://fossies.org/linux/misc/)

```bash
# 下载Fossies里当前最新版本
wget https://fossies.org/linux/misc/dialog-1.3-20251001.tgz

# 解压安装
sudo dnf install -y gcc make ncurses-devel
tar -xzf dialog-1.3-20251001.tgz
cd dialog-1.3-*
./configure
make
sudo make install

# 检查安装是否成功
dialog --version
```
